# Charter

## Purpose
- Capture stable engineering constraints for this repository.
- Keep this file durable; avoid session-specific notes or transient tasks.

## Product Constraints
- Define compatibility, rollout, and migration expectations here.

## Engineering Standards
- Prefer explicit, testable changes over clever shortcuts.
- Preserve backward compatibility unless a change is explicitly planned.
- Add or update tests when behavior changes.

## Review Gates
- Call out risks, operational impact, and validation before shipping.
